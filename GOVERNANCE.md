# Project Governance

## BDFL Model

This project operates under the **BDFL** (Benevolent Dictator For Life) model, currently held by the lead maintainer.

## Maintainers

Maintainers are responsible for:

- Reviewing and merging pull requests
- Triaging and responding to issues
- Releasing new versions to PyPI
- Maintaining CI/CD and infrastructure
- Enforcing the Code of Conduct
- Setting the project roadmap

See [MAINTAINERS.md](MAINTAINERS.md) for the current list.

## Becoming a Maintainer

1. Demonstrate sustained, high-quality contributions over at least 6 months.
2. Be nominated by an existing maintainer or self-nominate with evidence of contributions.
3. Achieve consensus approval from existing maintainers.
4. Accept the responsibilities outlined in MAINTAINERS.md.

## Decision Making

- **Technical decisions:** Consensus among maintainers. If consensus cannot be reached, the BDFL makes the final call.
- **Governance changes:** Require maintainer consensus.
- **New maintainers:** Require consensus approval (no veto by a single maintainer).

## Conflict Resolution

If a disagreement cannot be resolved through discussion, escalate to the BDFL. Serious or persistent violations of the Code of Conduct are handled according to the enforcement guidelines in CODE_OF_CONDUCT.md.

## Inactivity

Maintainers inactive for 6+ months may be moved to emeritus status. Emeritus maintainers can return to active status by resuming regular contributions.
